Anil

What I’ve done this week

  • Write Python / Matlab code to process data from M1 neural data in the monkey dataset
  • Write Python code to improve understanding of how the control signals coming from the DSP works in the context of the window counter circuit (ended up not completing and moved onto other things when I had a better understanding of it)
  • Feed processed data into ELM algorithm and test to see what results we’re getting
  • Play with hyperparameters such as p (the number of window counter delays, which also changes the input vector size for the ELM), D (the number of hidden nodes in the ELM), time_bins (the size of time bin used for counting neuron firings when processing the data).
  • Experimented with both multi-output regressional ELM (trying to predict x and y velocity using regression) and multi-class classification ELM types (directional detection discretized into n classes, I used n = {2, 4, 8} while experimenting) to evaluate performance metrics. Tried changing the number of classes to see its affect on classification accuracy.

My plan is to write another Matlab script that sweeps possible values for hyperparameters as well and evaluates performance and picks out the optimal hyperparameters when run for a long time (e.g. overnight).
